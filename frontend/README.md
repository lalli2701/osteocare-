# OsteoCare+

Contributor setup and run guide for the Flutter frontend and Flask backend.

## Project Structure

- `frontend/`: Flutter app
- `backend/`: Flask API used by the app

## Prerequisites

Install these before running locally:

- Flutter SDK (matching `sdk: ^3.11.0` in `pubspec.yaml`)
- Python 3.11+ (3.10+ should also work in most cases)
- Git
- Chrome (for Flutter web development)

Optional but recommended:

- Android Studio or VS Code Flutter extension

## 1. Clone Repository

```bash
git clone <your-repo-url>
cd osteocare-
```

## 2. Backend Setup (Flask)

From project root:

### Windows (PowerShell)

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### macOS/Linux

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Create `backend/.env` (optional for local dev; defaults exist):

```env
API_KEY=dev-key
PORT=5000
```

Run backend:

```bash
python app.py
```

Expected:

- API available at `http://localhost:5000`

## 3. Frontend Setup (Flutter)

Open a second terminal from project root:

```bash
cd frontend
flutter pub get
flutter doctor
```

Run on Chrome:

```bash
flutter run -d chrome
```

## 4. Local Integration Notes

- Frontend service URLs point to localhost (`http://localhost:5000` or `http://127.0.0.1:5000`) in current code, so run backend and frontend on the same machine.
- Keep backend running while using app features like auth, survey submit, dashboard, and chatbot.

## 5. Firebase Configuration

The app initializes Firebase in `lib/main.dart` using `lib/firebase_options.dart`.

- If your environment uses real Firebase, replace placeholder values in `lib/firebase_options.dart` with project values generated by FlutterFire.
- For contributor onboarding, confirm team policy for shared dev Firebase config before committing keys.

## 6. Common Issues

- `Connection refused` in app: backend is not running or wrong port.
- `flutter pub get` fails: run `flutter doctor` and fix missing SDK/toolchain items.
- Backend package install issues: ensure virtual environment is active before running `pip install -r requirements.txt`.

## 7. Contributor Workflow

1. Pull latest changes: `git pull`
2. Start backend terminal (`backend/`, virtual env active, `python app.py`)
3. Start frontend terminal (`frontend/`, `flutter run -d chrome`)
4. Make changes and test locally
5. Commit with clear message and open PR
